Part B: Password Security Review :-

1. What makes a strong password?
A strong password is long, unique, and difficult to guess. It should contain uppercase letters, lowercase letters, numbers, and special characters. It should not contain personal information like names or birthdays.

2. Why should passwords be unique?
Every account should have a different password. If one password is stolen, attackers cannot access your other accounts.

3. What is Multi-Factor Authentication (MFA)?
Multi-Factor Authentication (MFA) is an additional security method that requires users to verify their identity using two or more methods, such as a password and a one-time verification code sent to a phone.

4. What are the risks of password reuse?
Multiple accounts can be hacked.
Personal information may be stolen.
Financial loss can occur.
Identity theft becomes easier.
Hackers can perform credential stuffing attacks.

Password Policy (5 Best Practices)

Use at least 12–16 characters.
Include uppercase, lowercase, numbers, and symbols.
Never reuse passwords across different accounts.
Avoid personal information like birthdays or names.
Enable Multi-Factor Authentication whenever possible.

Part D – Cyber Threat Research :-

1. Phishing
   Definition
Phishing is a cyberattack where criminals send fake emails, messages, or websites to trick users into revealing sensitive information such as passwords or banking details.

   Real-world Example
A fake email pretending to be from a bank asks users to click a link and enter their login credentials.

   Prevention
Verify sender information.
Never click suspicious links.
Enable MFA.
Check website URLs carefully.
Use email spam filters.

3. Malware
   Definition
Malware is malicious software designed to damage computers, steal information, or disrupt normal operations.

   Real-world Example
A user downloads a fake software installer that secretly installs spyware.

   Prevention
Install antivirus software.
Download software only from trusted websites.
Keep the operating system updated.
Avoid suspicious email attachments.
Scan downloaded files.

3. Ransomware
   Definition
Ransomware encrypts files and demands payment to restore access.

   Real-world Example
The WannaCry ransomware attack infected thousands of computers worldwide.

   Prevention
Keep backups of important files.
Update software regularly.
Use antivirus protection.
Avoid opening unknown attachments.
Enable firewall protection.

5. Social Engineering
   Definition
Social engineering is the manipulation of people into revealing confidential information instead of exploiting technical vulnerabilities.

   Real-world Example
A scammer calls pretending to be IT support and asks for login credentials.

   Prevention
Verify identities.
Never share passwords.
Be cautious of urgent requests.
Educate users.
Report suspicious activities.

5. Data Breach
   Definition
A data breach occurs when unauthorized individuals gain access to confidential or sensitive information.

  Real-world Example
A company's customer database is stolen due to weak security.

  Prevention
Encrypt sensitive data.
Use strong passwords.
Enable MFA.
Limit access permissions.
Regularly update security systems.
